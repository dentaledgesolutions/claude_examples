<objective>
Call the live GHL MCP to fetch and cache two critical configuration datasets: (1) all custom field definitions for the location, and (2) the blog site ID, author ID, and default category ID needed for blog publishing.

Purpose: GHL custom field IDs are UUIDs assigned at account creation — they are not guessable or documented anywhere except the live API. /des-audit cannot map contact.customFields[].id to human-readable keys without this cache. Similarly, blogs_create-blog-post requires blogId, authorId, and categoryId that must be fetched once and stored. Both are one-time setup calls — cache them now so no skill ever needs to fetch them again.

Output: audit/field-map.json and knowledge/ghl-config.json, both committed to git.
</objective>

GHL MCP is configured in this project. The MCP server at services.leadconnectorhq.com/mcp/ is connected via the project's .mcp.json or ~/.claude/settings.json with a PIT token and locationId header.

Critical API behavior documented in research:

READ ASYMMETRY — GHL contact custom fields:
- GET response uses: { "id": "FIELD_UUID", "value": "..." }
- PUT payload uses: { "id": "FIELD_UUID", "field_value": "..." }
This asymmetry must be noted in field-map.json comments or README so /des-audit and /des-proposal implementers don't get it wrong.

GHL MCP tool names (verified against official spec):
- locations_get-custom-fields — GET /locations/{locationId}/customFields — returns all field definitions
- blogs_get-blogs — GET /blogs/site/all?locationId=X — returns blog site(s)
- blogs_get-all-blog-authors-by-location — GET /blogs/authors?locationId=X — returns authors
- blogs_get-all-categories-by-location — GET /blogs/categories?locationId=X — returns categories

Note: exact MCP tool names may vary slightly from the native GHL MCP server. If a tool call returns "tool not found", try alternate names:
- blogs_get-sites, blogs_list-sites for blog site lookup
- blogs_get-authors for author lookup
- blogs_get-categories for category lookup
Discover available blog tools by checking what mcp__ghl__* tools are listed in the current session.
</context>
